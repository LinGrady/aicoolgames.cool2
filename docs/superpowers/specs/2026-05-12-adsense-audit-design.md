# AdSense 全站深度体检设计

## 目标

本轮目标是把当前静态内容站按 Google AdSense 审核视角做一次完整体检，找出会影响过审的站点可信度、内容质量、技术合规和低价值信号问题，并形成可执行的修复顺序。

最终标准不是“页面看起来更丰富”，而是让站点更像一个真实、长期维护、单主题、单作者、内容有深度的个人理财出版站。

## 范围

### 1. 核心入口页

审查 `index.html` 与 `blog.html`。

重点判断：
- 首页是否清楚说明站点是谁写、写什么、为什么可信。
- 博客页是否按专题组织，而不是泛文章目录。
- MoneyMind / Jake / AI Cool Finance 等品牌和作者信息是否冲突。
- 是否存在广告占位、夸张数字、模板化营销话术。

### 2. 信任与合规页

审查 `about.html`、`contact.html`、`privacy-policy.html`、`terms-of-service.html`、`affiliate-disclosure.html`、`faq.html`、`resources.html`。

重点判断：
- 作者身份是否一致、可信、不过度包装。
- 联系方式、纠错机制、免责声明是否完整。
- Privacy / Terms / Affiliate Disclosure 是否覆盖 cookies、analytics、ads、第三方服务和内容责任边界。
- FAQ / Resources 是否有实质内容，避免空壳页或低价值页。

### 3. 文章页

审查 `blog/` 下所有文章。

重点判断：
- 标题、导语、段落结构是否过于模板化。
- 是否存在明显 AI 套话、重复段落、泛泛建议。
- 每篇文章是否有明确观点、实际权衡、具体场景和自然作者语气。
- 是否有合理内链和专题归属。

### 4. 站点基础文件

审查 `sitemap.xml`、`robots.txt`、`ads.txt`。

重点判断：
- sitemap 是否覆盖实际重要页面且无不存在页面。
- robots 是否允许正常抓取。
- ads.txt 是否与当前 AdSense 发布商 ID 一致。

## 问题分级

### P0：强过审风险

需要优先修复的问题：
- 广告占位或未通过审核前过强广告信号。
- 死链、空壳页、缺失合规页。
- 品牌、作者、站点名冲突。
- sitemap / robots / ads.txt 与实际站点不一致。
- 明显虚假或无法支撑的信任声明。

### P1：明显质量风险

应在 P0 后修复的问题：
- 首页和博客页缺少真实出版站结构。
- 文章 AI 感强、模板化、缺少个人判断。
- 专题组织弱，内链不足。
- About / Contact 等信任页空泛。

### P2：优化项

不阻塞第一轮修复，但建议后续优化：
- SEO title / description 更自然。
- 结构化数据更一致。
- 页面间推荐阅读更精细。
- 移动端可读性和视觉节奏优化。

## 执行策略

采用“先审查、再排序、再修复”的方式：

1. 扫描全站 HTML、sitemap、robots、ads.txt。
2. 记录所有 P0/P1/P2 问题。
3. 优先修复 P0，保证站点基础可信和可抓取。
4. 再修复 P1，提升内容站真实感和过审友好度。
5. 最后列出 P2，避免本轮过度设计。

## 非目标

本轮不做以下事情：
- 不引入 CMS、构建工具或前端框架。
- 不重做整套视觉设计。
- 不一次性重写所有文章。
- 不创建虚假作者资历、团队规模或无法验证的数据。
- 不为“看起来丰富”增加无实际价值页面。

## 验收标准

完成后应满足：
- 全站品牌与作者信息一致。
- 无明显广告占位、死链、空壳页。
- 关键合规页完整且口吻可信。
- sitemap、robots、ads.txt 与实际站点一致。
- 首页和博客页更像真实单作者出版站。
- 文章质量风险被分级，至少高风险问题有明确修复路径。
